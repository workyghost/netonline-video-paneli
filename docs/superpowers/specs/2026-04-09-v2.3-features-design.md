## 1. Orientation (Yön) Kaldırma

### Kapsam
- **dashboard.js**: Upload formundan yön select kaldır; içerikler listesinde yön filtresi ve badge kaldır; ekranlar tablosunda yön sütunu kaldır; ekran ekleme/düzenleme modallarından radio kaldır; `ORIENTATION_LABEL` / `ORIENTATION_BADGE` sabitleri kaldır.
- **player.html**: "Ekran Yönü" form alanını kaldır.
- **player.js**: `saveScreenButton` handler'dan orientation zorunluluğunu kaldır; DB insert'e `orientation: null` yaz (kolon kalır, kullanılmaz); `startFirmMode`'daki `.filter(v => v.orientation === ... || v.orientation === 'both')` satırını kaldır — tüm aktif videolar oynar.
- **DB schema**: `orientation` kolonu kalır — veri silinmez, uygulama katmanında kullanılmaz.

### Oynatma Davranışı
Yön filtresiz: firma modunda `is_active=true` ve `expires_at` geçmemiş tüm içerikler oynar. Blur fill zaten her en/boy oranını dolduruyor.

## 2. Görsel Desteği (jpg/png)

### Algılama Yöntemi
DB değişikliği yok. `isImage(fileName)` helper:
```js
const isImage = fn => /\.(jpg|jpeg|png)$/i.test(fn);
```
`file_name` uzantısına göre video/görsel ayrımı yapılır.

### Upload (dashboard.js)
- Dosya input `accept`: `.mp4,video/mp4,.jpg,.jpeg,.png,image/jpeg,image/png`
- Drop zone etiketi: "MP4 · JPG · PNG"
- Görsel yüklenince:
  - `thumbnail_url = file_url` (görselin kendisi, canvas adımı atlanır)
  - XHR upload aynı storage endpoint'e gider (mock server zaten görselleri bellekte saklar)
- İçerik listesinde görsel dosyaların önizlemesi zaten `thumbnail_url` ile gösterilir (değişiklik yok).

### Player (player.html + player.js)
**Yeni HTML elementler** (playerScreen içine, video elementlerinin yanına):
```html
<img id="bgImage"  class="absolute inset-0 w-full h-full object-cover blur-xl scale-110 opacity-60 hidden">
<img id="mainImage" class="absolute inset-0 w-full h-full object-contain z-10 hidden">
```

**`playVideo(index)` → `playItem(index)` olarak yeniden adlandırılır:**
- Video ise: mevcut davranış (`mainVideo.src`, `bgVideo.src`, play). `mainImage`/`bgImage` hidden.
- Görsel ise: `mainImage.src = url`, `bgImage.src = url`. Video elementleri durdurulup hidden. 30 saniye sonra `playNext()` çağrılır. Timer ref `imageTimer` state'e eklenir, `updateVideoQueue` / `playItem` çağrısında temizlenir.

**Heartbeat**: görsel gösterilirken `currentVideo` = görsel objesi (zaten mevcut state); `current_video_title` güncellenir.

**Hata dayanıklılığı**: görsel yüklenemezse (`mainImage.onerror`) → `consecutiveErrors++`, aynen video hatası gibi işlenir.

## 3. Ekranlar Listesi Anlık Güncelleme

### Sorun
`openAddScreenModal` / `openEditScreenModal`, insert/update sonrası yalnızca `closeModal()` + toast çağırıyor. VPS'te realtime gecikmesi olduğunda liste eskide kalıyor.

### Çözüm
`fetchScreens` fonksiyonunu `initScreens` içinden alıp `openAddScreenModal(fetchScreens)` ve `openEditScreenModal(dataset, fetchScreens)` parametresi olarak geçir. Başarılı insert/update sonrası hem `closeModal()` hem `fetchScreens()` çağrılır. Realtime subscription arka planda kalır (diğer client değişiklikleri için).

## 4. Kalıcı Hata Paneli

### Sorun
Error toast'lar 4 saniyede kayboluyor — kullanıcı mesajı okuyamıyor veya kopyalayamıyor.

### Çözüm
**İki toast davranışı:**
- `showToast(msg, "success")` → mevcut davranış (4s sonra kaybolur)
- `showToast(msg, "error")` → kalıcı toast, sağ altta birikir, her birinde ✕ butonu var, manuel kapatılana kadar durur

**Uygulama detayı (dashboard.js):**
- Mevcut `showToast` fonksiyonu güncellenir.
- Error toast container: `fixed bottom-4 right-4 flex flex-col gap-2 z-50` — üst üste binmeden yığılır.
- Her error toast'ta kopyalama kolaylığı için `select-all` stilinde küçük metin alanı veya sadece kapatma butonu.
- Toast ID ile yönetilir (aynı mesaj tekrarlanmaz).

## 5. Player Test Edilebilirliği (Mock Server)

Video dosyaları mock server'da saklanmıyor (drain ediliyor). Bu beklenen bir davranış; VPS'te gerçek storage çalışır.

**Lokal test için onay:** Mock server video URL'lerine (`/storage/v1/object/public/.../videos/*.mp4`) GET gelince 200 OK + minimal binary döndürür — tarayıcı "bozuk video" hatası verir ama `mainVideo.error` üç kez tetiklendikten sonra "İçerik yüklenemiyor" gösterir. Bu davranış kabul edilebilir; **lokal testte görsel (jpg/png) yükleme + oynatma test edilebilir** çünkü görseller bellekte saklanıyor.
